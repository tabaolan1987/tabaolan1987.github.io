Jmeter : How to Use Jmeter Test Database

How to use Jmeter test your Database

In this chapter we will see how to create a simple test plan to test the database server.

For our test purpose we have used the MYSQL database server. You can use any other database for testing.

Precondition

  1. Install MYSQL, create database and table in MYSQL, save user name and password can access to MYSQL.

  1. Download jar file for JMETER connect to MYSQL : Click her

  2. Extract file “com.mysql.jdbc_5.1.5.jar.zip”, you will see “com.mysql.jdbc_5.1.5.jar”. Rename this file to “com.mysql.jdbc.jar” and put it to folder lib of JMETER.

Create JMeter Test Plan

Open Jmeter from : apache-jmeter/bin/jmeter.bat

ADD USERS

Now create a Thread group, right click on Test Plan > Add> Threads(Users)> Thread Group.

Thread Group will get added under the Test Plan node. Rename this Thread Group as “USER CONNECT TO MYSQL”.

Set value for Numbers of Threads(users): 10

Set value for Ram-up Periods : 5

It means that in 5 seconds, we will set up 10 users connect to MYSQL server.

ADD JDBC Connection Configuration

Right click on the JDBC Users element, select Add > Config Element > JDBC Connection Configuration.

Set up the following fields (we are using MYSQL):

  1. Variable name bound to pool. This needs to uniquely identify the configuration. It is used by the JDBC Sampler to identify the configuration to be used. We have named it as “dbpooldemo” .

  2. Database URL: jdbc:mysql://youripserver:3306/yourdatabase

  3. JDBC Driver class: com.mysql.jdbc.Driver

  4. Username: username can access to your database created before.

  5. Password : password of this username.

ADD JDBC REQUESTS

Now that we have defined our users and configuration connection, it is time to define the tasks that they will be performing.

Select JDBC Users element, click your right mouse button to get the Add menu, and then select Add > Sampler > JDBC Request.

Then, select this new element to view its Control Panel. Edit the properties as below:

  1. Enter the Pool Name: dbpooldemo (same as in the configuration element)

  2. Query Type: Select statement

  3. Enter the SQL Query String field.

CREATE LISTENER

Now add the Listener element. This element is responsible for storing all of the results of your JDBC requests in a file and presenting a visual model of the data.

Select the JDBC Users element and add a View Results Tree listener (Add > Listener > View Results Tree).

SAVE AND EXECUTE TEST PLAN

Now save the above test plan as db_test.jmx. Execute this test plan using Run > Start option.
