Working With TestLink : How to Create a Project and Requirement in TestLink?

How to create a Project

  1. Go to “localhost/testlink” and enter username and password to login.

  2. Go to the Test Project Management section:

  1. Name Field : name of this project.

  2. Project Description : what is the aim of the Project, what is the target group, what is the business logic, what is the Test Environment.

  3. Requirements feature : we may specify requirements and see if they are well-covered by Test Cases.

  4. Testing priority – we may assign priority to Test Cases (high, medium, low).

  5. Test Automation – we may specify whether the test should be performed manually or automatically

  6. Inventory – Enable the inventory to this project.

3.You can now set this project here, in the top right corner

How to create a Requirement for Specific Project

  1. Choose a Project in the top right corner

  1. Go to the Requirement Specification in Requirement section:

  1. Select button Create for create requirement or select button Import for import the requirement.

  1. With Select button Create, it will look like this :

  1. Document ID : setup the id for this document.

  2. Title : write the title for this requirement.

  3. Scope : write the scope of this requirement.

  4. Type : select the type of this requirement such as : user or system requirement.

  5. After enter Save Button, you will see the Requirement is available in the left corner.
