Test Management Tool : Introduction

What is the Test Management Tool?

Test management tool actually is a website with 7 module are responsible for handle the testing process.

  1. The First module use to manage the Project for testing.

  2. The Second module use to manage the Requirement for specific Project created above, it also allow user can link test case with Requirement and manage the version of Requirement.

  3. The Third module use to manage the Test Design or Test Specification. It allow user can create test case, manage the version of each test case or link this test case with specific requirement.

  4. The Fourth module use to manage the Test Plan for each release of Software. It allow user can create a build for specific Test Plan and added a Test Suite for this plan. It also allow test lead can assign the test case for each tester.

  5. The Fifth module use to manage the Execution Test. It allow tester can update the result of each test case that assigned to him.

  6. The Sixth module use to manage the integration with Bug Tracker System. It allow tester can created a bug in Bug Tracker System and mapped with the specific test case.

  7. The Seventh module absolute to manage the Report for each Project, Test Plan or specific build with more information include Requirement, Test Result for each build,… etc!

Test Management Tool Use Case Diagram

  1. In this Diagram , I had created a basic use case diagram with : Guest, Tester, Test Lead, Project Manager (PM), Administrator. You can created more than use case base on the process of your company.

  2. With Guest, the use case diagram show that he can access to the Test Management tool and see the report only (In here Guest can be your client).

  3. With Tester, she have a permission for design test case, execute and update result for test case that assigned to her. She can assign the keywords also.

  4. With Test Leader, she have a permission for write of import the product requirement, she also can define test plan for each release and assign test case to requirement or assign test case to tester for execute update the result.

  5. With Project Manager , he have a permission for create specific project and assign whom can be join to this project.

  6. With Administrator, he have a permission for backup database and manage the user in the Test Management Tool.

Test Management Tool Process

  1. The first test to manage the testing process in Test Management Tool is create Project with some information to definition about this project.

  2. The next thing we should do is created or import the Requirement to this specific Project.

  3. Now it is the time for us to design test case and linked test case to the Requirement that we created above.

  4. After we have all test cased needed to cover 100% Requirement, We can create a Test Plan for the release of Software.

  5. In the Test Plan, we can create the build for this Test Plan.

  6. We have a Test Plan and the specific build , now we will created a test suite for this Test Plan by pick up test cases in step 2.

  7. Now the time we execute the test.

  8. During the time execute test if any test case fail, we will create issue and it will be tracked in the Bug Tracked System with mapping to this specific Test Case.

Test Management Tool integrate with Automation Test

Benefits when we apply the Test Management Tool

  1. We have all the documentation structured and organized.

  2. We solve the problem of version control.

  3. We can control the testing process (Events log + different kinds of Reports) .

  4. We can see if all the requirements are covered with Test Cases.

  5. We can select Test Cases for Regression Testing .

  6. We can bring the automation test and manual test together.

  7. We can see the results of testing in a very clear and easy to use form.
